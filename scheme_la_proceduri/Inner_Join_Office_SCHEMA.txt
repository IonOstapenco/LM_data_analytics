# 🟩 **2. PRELUCRARE – LOGICA PROCEDURII**

---

## **A. Identificare fișiere (cele mai recente)**

```
cm.list_files_scandir
     │
     ├─ FARM_MT*.xlsx → farm_file
     └─ ICTG*.csv → ictgc_file
```

✔ sortare descrescătoare
✔ se ia primul (cel mai recent)

---

## **B. Citire și normalizare**

```
FARM:
  DNSName → split(".")
           → prima parte
           → trim → UPPER
           → DNSName_processed

ICTG:
  Nome computer → trim → UPPER
```

---

## **C. Curățare și deduplicare**

```
FARM:
  [DNSName_processed, Gruppi di AD]

ICTG:
  [Nome computer, Nome componente]

→ drop_duplicates()
```

---

## **D. INNER JOIN (partea centrală)**

```
JOIN:
  DNSName_processed  =  Nome computer
```

✔ doar serverele care există în ambele fișiere

---

## **E. Rezultat inițial**

```
result_df:
  - Nome Server
  - Gruppi di AD
  - Nome componente
```

---

## **F. Excludere servere**

```
EXCLUDE:
  XA02MB2C
  XA11MA2C
```

---

## **G. Filtrare pe tip Office**

```
┌──────────────────────────────┐
│ Office Standard              │
│ contains "Office Standard"   │
└──────────────────────────────┘

┌──────────────────────────────┐
│ Office Professional          │
│ contains "Professional Plus" │
└──────────────────────────────┘
```

→ rezultă:

```
standard_df
professional_df
```

---

## **H. Extracție grupuri AD (regex)**

```
Pentru fiecare rând:
   Gruppi di AD → aplic regex
```

Exemple detectate:

```
CTX_121_XXX_PRD
Utenti CCM 099
UTENTI SDB 099
Sap 99
```

→ rezultat:

```
(gruppo, source)
  STD / PRO
```

---

## **I. Eliminare duplicate + logică prioritate**

```
1. eliminare duplicate
2. dacă apare în ambele:
      → păstrăm PRO
```

---

## **J. Filtru excluderi (keyword)**

```
EXCLUDE:
  TOOL
  YHC0062
  SAP
```

✔ doar dacă apare ca “cuvânt separat” (regex \b)

---

## **K. Variante pentru Excel (liste unice)**

```
Extrage grupuri diferite:
  CRxxxxx
  CRExxxx
  CTX_...
  Utenti ...
```

→ fără STD/PRO
→ doar liste unice
