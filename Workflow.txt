La chiave primaria è il nome del server e prima di poterla usare per una ricerca sui report del CMDB e/o del BigFix è necessario normalizzarla.
Normalizzarla significa, mettere tutte le lettere del nome nello stesso 'case' (maiuscolo o minuscolo) e, se presente, eliminare il dominio ( Es.: A27.ced.it => A27 )
Questa operazione di normalizzazione sarà eseguita nella procedura "Tabella HW" in fase di caricamento dei dati dai singoli report.

Da CMDB => Asset Server (ALL)-              ==>> CMDB -+ 
Da CMDB => Asset ServerDismessi (ALL)-      ==>> DISM  +---> tutti questi dati sono usati per scrivere ReportAsset_CMDB.csv
Da CMDB => Asset Client (ALL)-              ==>> PDL  -+
Da BigFix => ICTG-01-inventario-Hardware-   ==>> BFX  -----> ICTG-HW.csv

L'operazione di normalizzazione dovrà essere effettuata anche sui file:
        RvTools_tabvHost                    ==>> ESX  -----> Host_list.csv
        RvTools_tabvCluster                 ==>> CLS  -----> Cluster_list.csv
        RvTools_tabvInfo                    ==>> VM   -+
        RvTools_tabvCPU                     ==>> CPU   +---> VM_list.csv
        RvTools_tabvTools                   ==>> TOOL -+

    Il caricamento dei dati da RvTool genera una nuova struttura dati con i dati dei Vcenter  
                                            ==>> VCNTR ----> vCenter_list.csv
                                            
    La struttura dati VCNTR p necessaria per collezionare, in fase di caricamento dati da RvTool, i dati 
    specifici dei Vcenter. Per la precisione i valori:
            ==>> NumeroCores : il numero complessivo dei cores degli ESX amministrati da quel Vcenter
            ==>> NumeroHosts : il numero dei server ESX amministrati da quel Vcenter
            ==>> NumeroCluster : il numero dei cluster definiti e gestiti da quel Vcenter.
    I dati del file vCenter_list sono elaborati dalla procedura che genera Host_list.csv
    
    La procedura che genera Host_list.csv genera anche il file Cluster_list.csv
    
    La procedura che genera VM_list.csv legge, aggiorna e riscrive il file Cluster_list,csv generato in fase di elaborazione degli ESX.
    L'aggiornamento riguarda il numero delle VM, il numero delle VM attive (PoweredOn) e delle VM con Sistema operativo Windows.
    
        
Questi file devono essere recuperati nelle varie cartelle che raccolgono i dati estratti dai diversi vCenter attivi nella infrastruttura Cedacri.
